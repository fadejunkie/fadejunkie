## What Still Needs to Happen

### Content Deployment
The 7 content pages are written and SEO-optimized but need to be **published on wcorwin.com**. Options:

1. **iHouseWeb blog module** — if available, publish as blog posts. Requires activating the blog module and confirming URL structure.
2. **Standalone landing pages** — build as independent pages within iHouseWeb using the page editor.
3. **External content pages** — host on a subdomain or integrated platform. Less ideal for SEO (domain authority doesn't transfer as effectively).

**Recommendation:** Option 2 (standalone pages within iHouseWeb) is preferred. This keeps all content on the main domain and allows full control over URL slugs, meta tags, and schema.

### Internal Link Implementation
The linking map is ready — links need to be added to existing pages via the iHouseWeb page editor and built into new page templates.

### FAQPage Schema Deployment
Each new page needs page-specific FAQ schema injected. This can be done via the iHouseWeb "Extra Code for Head" field (per-page if available, or assembled for global injection with page conditionals).
