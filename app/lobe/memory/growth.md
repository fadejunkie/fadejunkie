# Lobe Growth Log

## Patterns Learned

### h2/h3 Section Label Violations
- **2026-03-29T13:00** — `components/PathSelector.tsx`: collapsible card header button contained `<h2>your paths</h2>` — developer acknowledged the inheritance trap in a comment but left it as h2 anyway. Converted to `<p>`. This confirms the violation exists in component files, not just page files, and that devs writing comments about known issues are still a risk surface — the fix must still be made.

### rgba(0,0,0,*) inline color violations
- **2026-03-29T07:30** — `app/page.tsx`: 6 standalone `rgba(0,0,0,*)` violations found in the light-mode social proof strip and "Three Paths" section header — none were in ternary dark-mode guards. Fixed: strip bg → `var(--accent)`, borders → `var(--border)`, stat label/eyebrow/subtitle colors → `var(--muted-foreground)`. Landing page inline styles are equal-risk to component files. Remaining `rgba(0,0,0,*)` hits in the same file are all in ternary expressions (already theme-aware) or shadow values — those are not violations.

### Raw button font-sans violations
- **2026-03-29T13:00** — `components/PathSelector.tsx`: collapsible card header `<button>` lacked `font-sans`; added alongside the h2→p fix in the same edit. Collapsible accordion-style card headers that use raw `<button>` wrappers are a confirmed dual-risk surface (h2 label AND missing font-sans in the same element).

### var(--font-geist-mono) ghost token violations
- **2026-03-29T09:15** — `components/manifesto.tsx`: 2× ghost token instances found — the attribution eyebrow "Shop Talk" and the PILLARS map label `<p>`, which renders 3× at runtime (1 attribution + 3 pillar labels = 4 exposures from 2 source lines). Pattern: components that use static data arrays (PILLARS, TESTIMONIALS, STATS) authored in one sitting are especially likely to propagate the ghost token to every array-rendered element simultaneously. A single `replace_all` fixes all instances. Landing-page editorial section components (Manifesto, Testimonials, hero) remain the highest-density surface for ghost token introduction.

### Inline style single-property violations (color token variant)
- **2026-03-29T18:00** — `app/(auth)/tools/page.tsx`: 2× `style={{ color: "var(--link)" }}` on eyebrow `<p>` labels ("Shop" eyebrow on the Website Builder card, and the `{tool.category}` eyebrow on each tool card). The `--color-link` token is registered in `globals.css @theme inline` as `var(--link)`, making `text-link` a valid Tailwind utility that resolves identically. The inline style bypassed className audits entirely. Pattern confirmed: **devs who know the exact CSS variable name for a color reach for `style={{ color: "var(--token)" }}` instead of checking @theme inline for the utility class**. The correct workflow is always: (1) check if the token appears in `@theme inline` with `--color-` prefix, (2) if yes, use the Tailwind utility class (`text-link`, `text-muted-foreground`, etc.), (3) only use inline style if the token has no utility mapping. Detection grep: `style=.*color.*var(--` across all TSX files.

### Inline style FULL-BLOCK violations — pure display components
- **2026-03-29T19:15** — `components/PublicStatusBadges.tsx`: 3× complete inline style blocks across all element types in the component — compact badge `<span>`, path group label `<p>`, and full badge `<span>`. Every property in all three blocks had a direct Tailwind equivalent: `fontFamily` → `font-mono`, `fontSize` → `text-[N]`, `color` → `text-foreground` / `text-muted-foreground`, `opacity` → `/60` modifier on text color, `border` with `color-mix` → `border border-foreground/[18%]`, `borderRadius: "2px"` → `rounded-[2px]`, `padding` → padding utilities, `letterSpacing` → `tracking-[N]`. After fix: zero `style={{}}` props anywhere in the file. **Key insight: pure display components (no "use client", no hooks, props-in → JSX-out) are the fastest-authored and least-audited files in the codebase.** They're scaffolded in a single sitting without referencing the Tailwind token table, and then never opened again because they're visually simple and "just work." The full inline style block is the tell — not just one property, but ALL typography and layout properties for an element delivered as one `style={{}}` object. Detection: `grep -l 'style=\{\{' components/` filtered to files without `"use client"` — every hit deserves a full audit pass.
