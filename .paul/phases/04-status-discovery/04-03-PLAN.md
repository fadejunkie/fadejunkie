<objective>
## Goal
Add a lightweight connection flow to the status ecosystem — when a user discovers someone through browse or matching, they can express interest by sending a connection request tied to a specific status. Build the schema, backend, and task Lobe with the UI.

## Purpose
Discovery and matching (04-01, 04-02) let users *find* each other. But finding without a way to act is a dead end. This plan closes the loop: a user sees a relevant status, taps "Connect," optionally writes a short note, and the other user sees the incoming request. This is intentionally not a messaging system — it is a lightweight signal of interest that gives the recipient enough context to decide whether to follow up outside the platform (phone, Instagram, in-person). The connection is the bridge between discovery and real-world action.

## Output
- `statusConnections` table added to `app/convex/schema.ts` — tracks connection requests between users, linked to a specific status
- `connectOnStatus` mutation and `getMyConnectionRequests` query added to `app/convex/statuses.ts`
- Lobe task brief at `lobe/inbox/status-connections-ui.md` for a "Connect" button on discovery/match cards and a connections inbox
</objective>

<task type="auto">
  <name>Task 2: Add connection mutations and queries to statuses.ts</name>
  <files>app/convex/statuses.ts</files>
  <action>
    Add three new exports to `app/convex/statuses.ts`, placed in a new section after the `// ── Discovery ──` section and before `// ── Internal (cron-only) ──`. Use the section comment `// ── Connections ──`.

    **1. `connectOnStatus` mutation**

    Args:
    - `statusId`: `v.id("statuses")`
    - `note`: `v.optional(v.string())`

    Implementation:
    1. Get authenticated userId via `getAuthUserId`. If not authenticated, throw `ConvexError("Not authenticated")`.
    2. Look up the target status by ID. If it does not exist, throw `ConvexError("Status not found")`.
    3. Validate the status is active and not expired (`isActive === true` and `expiresAt > Date.now()`). If not, throw `ConvexError("Status is no longer active")`.
    4. Validate the user is not connecting to their own status (`status.userId !== userId`). If so, throw `ConvexError("Cannot connect to your own status")`.
    5. Check for existing connection: query `statusConnections` with `by_fromUserId_statusId` index matching `fromUserId === userId` and `statusId === statusId`. If a record exists, throw `ConvexError("Already sent a connection request for this status")`.
    6. Trim the note if provided. Cap at 280 characters: `note?.trim().slice(0, 280)`.
    7. Insert into `statusConnections`:
       ```ts
       {
         fromUserId: userId,
         toUserId: status.userId,
         statusId,
         note: trimmedNote || undefined,
         status: "pending",
         createdAt: Date.now(),
       }
       ```
    8. Return the new connection ID.

    **2. `getMyConnectionRequests` query**

    Args: none (uses authenticated user context)

    Implementation:
    1. Get authenticated userId. If not authenticated, return empty array.
    2. Query `statusConnections` with `by_toUserId` index matching the current user's ID. Collect all.
    3. For each connection, look up the linked status by `statusId` (via `ctx.db.get()`). Extract `path` and `toggleKey` (or null if the status was deleted — unlikely but defensive).
    4. Batch-lookup barber profiles for unique `fromUserId` values — same `barberMap` pattern used in `discoverStatuses`:
       - Collect unique `fromUserId` values
       - Query `barbers` table via `by_userId` index for each
       - Build a map of `{ name, slug, avatarUrl }`
    5. Return enriched results sorted by `createdAt` descending:
       ```ts
       {
         _id, // connection ID
         fromUserId,
         statusId,
         note,
         status, // "pending" | "seen"
         createdAt,
         statusPath: string | null,
         statusToggleKey: string | null,
         barberName: string | null,
         barberSlug: string | null,
         barberAvatarUrl: string | null,
       }
       ```

    **3. `markConnectionSeen` mutation**

    Args:
    - `connectionId`: `v.id("statusConnections")`

    Implementation:
    1. Get authenticated userId. If not authenticated, throw `ConvexError("Not authenticated")`.
    2. Look up the connection by ID. If not found, throw `ConvexError("Connection not found")`.
    3. Validate `connection.toUserId === userId`. If not, throw `ConvexError("Not your connection")`.
    4. Patch the connection: `{ status: "seen" }`.

    Do NOT modify any existing mutations or queries. Do NOT change imports except to add what is needed (no new imports should be necessary — `v`, `ConvexError`, `getAuthUserId`, `query`, `mutation` are already imported).
