<acceptance_criteria>

## AC-1: User can activate a status toggle
```gherkin
Given an authenticated user with a known path (e.g., "barber")
When they call activateStatus with path="barber" and toggleKey="seeking_employment"
Then a new status record is created with isActive=true
And activatedAt is set to the current timestamp
And expiresAt is calculated from the toggle's default_days
And if a status with the same userId+toggleKey already exists and is active, it returns an error
```

## AC-2: User can deactivate a status toggle
```gherkin
Given a user with an active status
When they call deactivateStatus with the status ID
Then isActive is set to false
And archivedAt is set to the current timestamp
```

## AC-3: User can refresh an active status
```gherkin
Given a user with an active status that is approaching expiration
When they call refreshStatus with the status ID and optional custom days
Then expiresAt is recalculated from now using default_days (or custom days up to max_days)
And refreshCount is incremented
```

## AC-4: Queries return correct status data
```gherkin
Given multiple users with various active and inactive statuses
When getMyStatuses is called by an authenticated user
Then it returns only that user's statuses (active and recent archived)

When getActiveByPath is called with path="barber"
Then it returns only active statuses for that path across all users
```

  <files>app/convex/statuses.ts</files>
  <action>
    Create `app/convex/statuses.ts` with authenticated mutations using `@convex-dev/auth`:

    1. **activateStatus** mutation:
       - Args: path (string), toggleKey (string)
       - Auth: require authenticated user (use getAuthUserId from @convex-dev/auth/server)
       - Validate: path is a valid UserPath, toggleKey exists for that path (use getToggleConfig)
       - Check: no existing active status with same userId + toggleKey (query by_userId_toggleKey index)
       - If duplicate active: throw ConvexError("Status already active")
       - Calculate expiresAt: now + (config.default_days * 86400000)
       - Insert: { userId, path, toggleKey, isActive: true, activatedAt: Date.now(), expiresAt, refreshCount: 0 }
       - Return the new status ID

    2. **deactivateStatus** mutation:
       - Args: statusId (v.id("statuses"))
       - Auth: require authenticated user
       - Validate: status exists and belongs to the authenticated user
       - Patch: { isActive: false, archivedAt: Date.now() }

    3. **refreshStatus** mutation:
       - Args: statusId (v.id("statuses")), customDays (v.optional(v.number()))
       - Auth: require authenticated user
       - Validate: status exists, belongs to user, isActive is true
       - Get toggle config for the status's path + toggleKey
       - Calculate new expiresAt: now + (days * 86400000)
         - days = min(customDays ?? config.default_days, config.max_days)
       - Patch: { expiresAt: newExpiresAt, refreshCount: (current.refreshCount ?? 0) + 1 }

    Import patterns from existing Convex files (e.g., app/convex/barbers.ts) for auth pattern.
    Import { getToggleConfig, USER_PATHS } from "./statusConfig" for validation.
    Use ConvexError from "convex/values" for user-facing errors.
  </action>

       - Args: path (string)

       - Args: userId (v.id("users"))
       - No auth required (public — for viewing other users' profiles)
       - Query statuses by_userId index
       - Filter to isActive=true and not expired
       - Return active statuses
