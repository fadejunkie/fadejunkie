<task type="auto">
  <name>Task 1: Add statuses table to Convex schema</name>
  <files>app/convex/schema.ts</files>
  <action>
    Add a `statuses` table to the existing schema. Do NOT modify any existing tables.

    Fields:
    - userId: v.id("users") — who owns this status
    - path: v.string() — user path (barber, student, shop, etc.)
    - toggleKey: v.string() — specific toggle (seeking_employment, hiring_barbers, etc.)
    - isActive: v.boolean() — current state
    - activatedAt: v.number() — timestamp when activated
    - expiresAt: v.number() — timestamp when it auto-expires
    - archivedAt: v.optional(v.number()) — timestamp when archived (null if active or expired)
    - refreshCount: v.optional(v.number()) — how many times manually refreshed

    Indexes:
    - by_userId: ["userId"] — get all statuses for a user
    - by_path_active: ["path", "isActive"] — get all active statuses for a path
    - by_userId_toggleKey: ["userId", "toggleKey"] — check if user has a specific toggle
    - by_expiresAt: ["expiresAt"] — for the expiration cron to find expired statuses

    Place the table BEFORE the Control Center tables comment block.
    Do NOT touch any existing table definitions.
  </action>
  <verify>Run `npx convex dev --once` from app/ to validate schema compiles</verify>
  <done>AC-1 satisfied: statuses table exists with all fields and indexes</done>
</task>
