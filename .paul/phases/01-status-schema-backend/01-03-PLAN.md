<objective>
## Goal
Build the automatic expiration engine — a Convex cron job that finds expired statuses and archives them, plus a manual archive-all helper.

## Purpose
Statuses have time-limited lifespans (1-90 days). Without automatic expiration, stale statuses pollute the discovery feed and degrade trust. The cron ensures the ecosystem stays fresh and accurate.

## Output
- Cron job added to `app/convex/crons.ts`
- `expireStatuses` internal mutation in `app/convex/statuses.ts`
</objective>

## AC-1: Expired statuses are automatically archived
```gherkin
Given statuses exist where expiresAt < current time and isActive = true
When the cron job runs
Then those statuses have isActive set to false and archivedAt set to current timestamp
And statuses not yet expired remain untouched
```

## AC-2: Cron runs on a reasonable schedule
```gherkin
Given the crons.ts file is deployed
When Convex processes the cron schedule
Then the expiration check runs every hour
```

## AC-3: Batch processing handles scale
```gherkin
Given 100+ expired statuses exist
When the cron runs
Then all expired statuses are processed (not just the first N)
```

    Add to the existing `app/convex/statuses.ts`:

    1. Import `internalMutation` from "./_generated/server"
    2. Create **expireStatuses** as an internalMutation (no auth needed — called by cron):
       - Query statuses using by_expiresAt index
       - Filter: isActive === true AND expiresAt < Date.now()
       - For each expired status: patch { isActive: false, archivedAt: Date.now() }
       - Return count of archived statuses
       - Use .collect() — Convex handles pagination internally

    This is an internal mutation — NOT exposed to clients. Only the cron calls it.

<task type="auto">
  <name>Task 2: Create crons.ts with hourly expiration schedule</name>
  <files>app/convex/crons.ts</files>
  <action>
    Check if `app/convex/crons.ts` already exists. If it does, ADD to it. If not, create it.

    Create `app/convex/crons.ts`:
    ```
    import { cronJobs } from "convex/server";
    import { internal } from "./_generated/api";

    const crons = cronJobs();

    crons.interval(
      "expire stale statuses",
      { hours: 1 },
      internal.statuses.expireStatuses
    );

    export default crons;
    ```

    This runs expireStatuses every hour. The interval is a good balance between freshness and resource usage — statuses with 1-day lifespans get archived within an hour of expiry.
  </action>
  <verify>Run `npx convex dev --once` and confirm cron is registered in output</verify>
