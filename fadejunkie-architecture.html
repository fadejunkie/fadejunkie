Agent Roster — V2 Orchestration

🎯 Dispatch opus execute

  • Role: Orchestrator — decomposes high-level tasks into atomic subtasks
  • Routes to: All 6 specialist agents based on task type
  • Manages: Dependency chains, manifests, escalations
  • Escalation: Ambiguous decisions → dispatch/escalations/ → Anthony
  • CWD: Workspace root

🧠 Funkie sonnet plan

  • Role: Strategy, product decisions, Q1 goals, client relationships
  • Default mode: Plan — writes to outbox/pending/ for review
  • Trust: LOW — always plan mode unless updating context/goals
  • Memory: funkie/memory/goals.md (Q1 2026 goals)
  • CWD: app/

🎨 Lobe sonnet execute

  • Role: ALL frontend/UI code — components, design, visual polish
  • Trust: HIGH — auto-execute, trusted to deliver without review
  • Tools: 21st.dev API, Framer API, shadcn/ui, Tailwind 4
  • Rule: Lobe owns all HTML/CSS/React. No exceptions, even one-liners.
  • CWD: app/ (overridable with project: header)

⚙️ Convex Agent sonnet execute

  • Role: Backend — schema, mutations, queries, auth, data modeling
  • Trust: MEDIUM — auto-execute additions, plan for destructive ops
  • Schema: Read-only by default. Additive changes OK.
  • Git protocol: Snapshots before + descriptive commits after
  • CWD: app/ (where convex/ lives)

✍️ Ink opus execute

  • Role: All customer-facing words — copy, proposals, contracts, social
  • Trust: HIGH — auto-execute, delivers without review
  • Voice: Direct, punk, culture-native. Not corporate. Not safe.
  • Per-client: loads custom voice context
  • CWD: ink/

📈 SEO Engine opus plan

  • Role: SEO audits, keyword research, strategy, client deliverables
  • Trust: MEDIUM — plan mode for client-facing changes
  • Max turns: 40 (higher for deep analysis)
  • Clients: Weichert Realtors — Corwin & Associates ($950/mo)
  • Rule: Does NOT edit Next.js code. Produces briefs for Funkie.

🛡️ Sentinel sonnet execute

  • Role: QA — build verification, visual tests, deploy gate
  • Trust: AUTO — always execute, QA is always action
  • Protocol: Build → Lint → Visual QA → Verdict (SHIP or FIX)
  • Failure: Same agent fails twice → escalate to Dispatch
  • Tools: Playwright for browser automation & screenshots

Task Lifecycle — Drop to Deploy
1

Task Arrives

Anthony drops a high-level .md task brief into dispatch/inbox/. Task headers control mode, model, turn limits, and dependency chains.

Anthony → Dispatch
2

Dispatch Decomposes

Dispatch reads the task, identifies required specialties, creates atomic subtasks, sets dependency order, and writes a manifest to dispatch/outbox/.

Dispatch
3

Subtasks Route to Specialists

Each subtask drops into the right agent's inbox/. Trust level determines execute vs. plan mode. Dependencies are respected — Lobe waits for Convex if backend is needed first.

Funkie Lobe Convex Ink SEO Engine
4

Agents Execute

Each agent runs via its REPL: reads inbox → executes task → writes code/copy/strategy → commits to git → writes result to outbox/. Plan-mode agents write to outbox/pending/ for review.

Lobe writes code Convex writes functions Ink writes copy
5

Sentinel QA Gate

Sentinel runs the QA protocol: npm run build → npm run lint → Playwright visual comparison → writes verdict. SHIP or FIX REQUIRED.

Sentinel
6

Resolution or Escalation

If SHIP → done. If FIX → Sentinel writes fix-task back to the originating agent. If same agent fails twice → escalate to dispatch/escalations/ for Anthony to review.

Sentinel Dispatch
