# Template: Milestone Update

## Structure

1. **Greeting** — Hey {first names} (use first names from client config contacts)
2. **What was just completed** — Specific deliverables, not vague ("10 unique title tags" not "SEO work")
3. **Measurable results** — Metrics, before/after numbers if available (clicks, impressions, pages)
4. **What's still wrapping up** — Only if something is genuinely in progress (omit if nothing)
5. **What's coming next** — Preview the next milestone (keep it short, builds anticipation)
6. **Project hub link** — "You can always see the latest at {hub URL}"
7. **CTA** — "Any questions? Happy to jump on a quick call."
8. **Sign-off** — Best, Anthony

## Rules

- Be specific about deliverables — list them, don't summarize
- Include numbers when available — let data speak
- Keep it scannable — bullet points for lists of 3+ items
- Tone matches client config (warm, professional, results-focused vs. energetic, milestone-driven)
- Never oversell — state facts, let results speak
- 150-300 words is the sweet spot — respect their inbox
- No jargon unless the client has used it themselves
- Subject line format: "Progress Update — {Milestone Name}"
